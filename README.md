# Flow-Matching Speech Enhancement via Dynamic Multi-Layer Tokenizer Alignment

> **PoC Academic Project** — Course project exploring generative models for speech enhancement under strict compute constraints (Google Colab Pro).

## Overview

This project implements a **Rectified Flow (Flow-Matching)** approach to speech enhancement in the continuous latent space of the [DAC (Descript Audio Codec)](https://github.com/descriptinc/descript-audio-codec). The generative process is conditioned on representations extracted from the **1.6B MOSS-Audio-Tokenizer**.

### Core Innovation

Instead of conditioning only on the last hidden layer of the MOSS tokenizer, we inject **dynamic multi-layer features** — capturing both semantic/prosody information (deep layers) and acoustic details (shallow layers) — via Cross-Attention in a Diffusion Transformer (DiT). We evaluate this through a controlled **ablation study** comparing three conditioning strategies.

### Architecture Diagram

```
Noisy Audio ──► DAC Encoder ──► X_0 (50Hz latent) ──┐
                                                      ├──► DiT (Rectified Flow) ──► X̂_1 ──► DAC Decoder ──► Enhanced Audio
Noisy Audio ──► MOSS Tokenizer ──► C (12.5Hz embed) ─┘
                                   (multi-layer)
```

## Pipeline — Step by Step

The pipeline has **4 stages**, each designed to run independently. Feature extraction (Steps 2–3) is done **offline** so that training is fast and memory-efficient.

### Step 1: Mix Clean Speech + Noise

Place clean speech files in `data/raw/clean/` and noise files in `data/raw/noise/`, then run:

```bash
python -m src.data.mixer \
    --clean_dir data/raw/clean \
    --noise_dir data/raw/noise \
    --out_dir   data/mixed \
    --snr_list  0 5 10 \
    --sr        16000
```

This creates `data/mixed/snr_0dB/`, `data/mixed/snr_5dB/`, `data/mixed/snr_10dB/` with mixed noisy audio files.

### Step 2: Extract DAC Latents (Offline)

Extract continuous latents (50 Hz) from both clean and noisy audio:

```bash
# Clean speech DAC latents
python -m src.data.extract_dac \
    --audio_dir data/raw/clean \
    --out_dir   data/features/clean_dac \
    --device    cuda

# Noisy speech DAC latents (pick an SNR level)
python -m src.data.extract_dac \
    --audio_dir data/mixed/snr_5dB \
    --out_dir   data/features/noisy_dac \
    --device    cuda
```

### Step 3: Extract MOSS Embeddings (Offline)

```bash
# Last-layer only (for Baseline 2)
python -m src.data.extract_moss \
    --audio_dir data/mixed/snr_5dB \
    --out_dir   data/features/moss_last \
    --device    cuda

# All hidden layers (for "Ours" multi-layer ablation)
python -m src.data.extract_moss \
    --audio_dir data/mixed/snr_5dB \
    --out_dir   data/features/moss_multi \
    --device    cuda \
    --save_all_layers
```

### Step 4: Training

Train any of the three ablation variants by setting `--condition_type`:

```bash
# Baseline 1: No conditioning
python train.py --config configs/default.yaml --condition_type none

# Baseline 2: MOSS last-layer conditioning
python train.py --config configs/default.yaml --condition_type last_layer

# Ours: MOSS dynamic multi-layer conditioning
python train.py --config configs/default.yaml --condition_type multi_layer
```

Checkpoints are saved to `checkpoints/<condition_type>/`.

### Step 5: Evaluation

```bash
python evaluate.py \
    --config configs/default.yaml \
    --checkpoint checkpoints/multi_layer/step_50000.pt \
    --condition_type multi_layer
```

This will:
1. Run the ODE solver (Euler method) to generate enhanced latents.
2. Decode latents back to waveforms via DAC decoder.
3. Compute FAD against clean reference audio.
4. Save Mel-spectrogram comparison plots.

## Ablation Study

We compare **three model variants** to isolate the effect of multi-layer conditioning:

| Variant | `condition_type` | Description |
|---------|------------------|-------------|
| **Baseline 1** | `none` | Vanilla DiT — only noisy DAC latent + timestep (no MOSS conditioning) |
| **Baseline 2** | `last_layer` | DiT + Cross-Attention with MOSS **last hidden layer** only |
| **Ours** | `multi_layer` | DiT + Cross-Attention with **learnable weighted-sum of multiple MOSS layers** |

### Hypothesis

Multi-layer features from MOSS capture complementary information:
- **Deep layers** → high-level semantic and prosody features
- **Shallow layers** → fine-grained acoustic details

By dynamically weighting these layers (via learned softmax weights), the model can attend to the most relevant features at each denoising step, leading to better reconstruction quality.

### Key Design Decisions

1. **Offline feature extraction** — DAC and MOSS encoders are never run during training, keeping GPU memory and compute usage minimal.
2. **Rectified Flow** — Straight-path ODE from noisy to clean, with a simple MSE loss on the vector field. No noise scheduling or complex diffusion processes.
3. **adaLN for timestep injection** — Standard adaptive layer normalisation, proven effective in DiT architectures.
4. **Learnable layer weights** — A single `nn.Parameter` vector with softmax normalisation controls the contribution of each MOSS layer.

## Evaluation Metrics

### FAD (Fréchet Audio Distance)

FAD measures the distributional similarity between generated and reference audio in a learned embedding space. Lower is better.


## Training Objective

We use **Rectified Flow** with straight-path interpolation:

$$X_t = t \cdot X_1 + (1 - t) \cdot X_0$$

where $X_0$ is the noisy DAC latent and $X_1$ is the clean DAC latent.

The target vector field is constant along the path:

$$V = X_1 - X_0$$

The training loss is:

$$\mathcal{L} = \mathbb{E}_{t \sim U(0,1)} \left[ \| v_\theta(X_t, t, C) - V \|^2 \right]$$

At inference, we solve the ODE $\frac{dx}{dt} = v_\theta(x_t, t, C)$ from $t=0$ to $t=1$ using the Euler method.

## Configuration

All hyperparameters are in `configs/default.yaml`. Key settings:

| Parameter | Default | Description |
|-----------|---------|-------------|
